详细描述

触发条件

事件的触发方式可通过参数 trigger 设置。事件的触发方式分为两类。

电平触发

  • XWOS_FLG_TRIGGER_SET_ALL : 掩码中的所有位同时为1
  • XWOS_FLG_TRIGGER_SET_ANY : 掩码中的任意位为1
  • XWOS_FLG_TRIGGER_CLR_ALL : 掩码中的所有位同时为0
  • XWOS_FLG_TRIGGER_CLR_ANY : 掩码中的任意位为0

电平触发 时，若参数 action 取值 XWOS_FLG_ACTION_CONSUMPTION ，表示线程在读取事件位图后，会 清除 事件。 电平触发 时，参数 origin 用于返回事件发生时的位图状态。

边沿触发

  • XWOS_FLG_TRIGGER_TGL_ALL : 掩码中的所有位同时发生翻转
  • XWOS_FLG_TRIGGER_TGL_ANY : 掩码中的任意位发生翻转

边沿触发 时，参数 action 没有作用，取值 XWOS_FLG_ACTION_NONE 即可。 边沿触发 时，必须要有一个初始状态，就像数字电路一样：

  • 当位的初始值为 0 (低电平)，然后跳变到 1 (高电平)的瞬间被称为上升沿。此时触发的事件被称为上升沿触发。
  • 当位的初始值为 1 (高电平)，然后跳变到 0 (低电平)的瞬间被称为下降沿。此时触发的事件被称为下降沿触发。

初始值就由参数 origin 指明，触发后，事件位发生跳变，跳变后的结果还是由 参数 origin 返回。

XWOS API：检查触发事件

参数
[in]flg事件标志对象指针
[in]trigger事件触发条件，取值：
  • 电平触发
    • XWOS_FLG_TRIGGER_SET_ALL : 掩码中的所有位同时为1
    • XWOS_FLG_TRIGGER_SET_ANY : 掩码中的任意位为1
    • XWOS_FLG_TRIGGER_CLR_ALL : 掩码中的所有位同时为0
    • XWOS_FLG_TRIGGER_CLR_ANY : 掩码中的任意位为0
  • 电平触发
    • XWOS_FLG_TRIGGER_TGL_ALL : 掩码中的所有位同时发生翻转
    • XWOS_FLG_TRIGGER_TGL_ANY : 掩码中的任意位发生翻转
[in]action事件触发后的动作，
  • 当trigger为 电平触发 时，取值：
    • XWOS_FLG_ACTION_CONSUMPTION : 消费事件
    • XWOS_FLG_ACTION_NONE : 无操作
