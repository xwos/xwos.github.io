中断控制

XWOS的中断控制
Categories:
  • XWOS

  2 分钟阅读  

约束

异常与中断

XWOS将中断分为EXC和IRQ：

  • EXC是指架构中定义的中断，在某些架构中被称为 异常(Exception) ，中断号用负数表示；
  • IRQ是指外设中断，例如UART的中断等，中断号用0和正数表示。

中断优先级

XWOS对中断优先级的要求：

切换上下文的中断 <= 滴答定时器的中断 < 调度器服务中断
切换上下文的中断 < 其他中断 < 调度器服务中断

中断向量表

中断向量表通常位于 ivt.c 文件中，此文件可能存放在

  • xwbd/电路板名称/bdl；
  • xwbd/电路板名称/bm 下的某个玄武模块。

例如，XWOS自带的工程，中断向量表位于：

  • xwbd/WeActMiniStm32H750Bootloader/bm/stm32cube/cubemx/IVT/ivt.c
  • xwbd/WeActMiniStm32H750/bm/stm32cube/cubemx/IVT/ivt.c
  • xwbd/EmbedFireStm32H743/bm/stm32cube/cubemx/IVT/ivt.c
  • xwbd/ATKStm32H743/bm/stm32cube/cubemx/IVT/ivt.c
  • xwbd/FK429M1/bm/stm32cube/cubemx/IVT/ivt.c
  • xwbd/MPC5607BCore/bdl/ivt.c

XWOS要求给出下列形式的中断向量的定义：

__xwos_ivt __xwos_ivt_qualifier struct soc_ivt xwospl_ivt;
__xwos_ivt_qualifier struct soc_idvt xwospl_idvt;

  • __xwos_ivt ：中断向量的段落属性 .xwos.ivt ，链接脚本通过此属性，识别中断向量，并在链接时将它安排在特定的区域内。

  • __xwos_ivt_qualifier ：中断向量的读写修饰符，与 xwbd/电路板/cfg/soc.h 中配置有关：

    • SOCCFG_RO_IVT 配置为1： __xwos_ivt_qualifier 为 const ，表示只读，位于ROM内；
    • SOCCFG_RO_IVT 配置为0： __xwos_ivt_qualifier 为空，表示可读写，位于RAM内。

  • struct soc_ivt ： 中断向量表结构体类型 ，为某个芯片移植XWOS时，需要在操作系统移植实现层(XWOSIMPL)的 xwosimpl_soc_irq.h 文件中定义此结构体。

  • xwospl_ivt ：上述结构体的变量；

  • struct soc_idvt ： 中断数据表结构体类型 ，为某个芯片移植XWOS时，需要在操作系统移植实现层(XWOSIMPL)的 xwosimpl_soc_irq.h 文件中定义此结构体， 此结构体用于给每个中断存放私有数据；

  • xwospl_idvt：上述结构体的变量。

各种架构的实现举例：

  • ARMv6m、ARMv7m的定义如下：
struct soc_ivt {
        xwisr_f exc[SOCCFG_EXC_NUM];
        xwisr_f irq[SOCCFG_IRQ_NUM];
};

struct soc_idvt {
        void * exc[SOCCFG_EXC_NUM];
        void * irq[SOCCFG_IRQ_NUM];
};
  • RISC-V/nuclei/gd32v的定义如下：
struct soc_ivt {
        __xwos_ivt_qualifier soc_esr_f * exc;
        __xwos_ivt_qualifier xwisr_f * irq;
};

struct soc_idvt {
        __xwos_ivt_qualifier struct soc_irq_data * exc;
        __xwos_ivt_qualifier struct soc_irq_data * irq;
};
  • PowerPC/e200z0h/mpc560xb的定义如下：
struct soc_ivt {
        xwisr_f irq[SOCCFG_IRQ_NUM];
};

struct soc_idvt {
        void * irq[SOCCFG_IRQ_NUM];
};

CPU中断开关

操作CPU的中断开关的功能，是XWOS内核比较重要的依赖。 CPU只能操作自身的中断开关，不可操作其他CPU的中断。下面的函数运行在哪个CPU，就对哪个CPU起作用：

  • xwos_cpuirq_enable_lc() ：开启CPU的中断开关
  • xwos_cpuirq_disable_lc() ：关闭CPU的中断开关
  • xwos_cpuirq_save_lc() ：保存当前CPU的中断开关状态后关闭
  • xwos_cpuirq_restore_lc() ：恢复之前保存的CPU的中断开关状态

为某个芯片移植XWOS时，需要在操作系统移植实现层(XWOSIMPL)中给出函数的实现方法：

  • 源文件： xwosimpl_irq.h/xwosimpl_irq.c
    • xwospl_cpuirq_enable_lc() ：对应于 xwos_cpuirq_enable_lc()
    • xwospl_cpuirq_disable_lc() ：对应于 xwos_cpuirq_disable_lc()
    • xwospl_cpuirq_save_lc() ：对应于 xwos_cpuirq_save_lc()
    • xwospl_cpuirq_restore_lc() ：对应于 xwos_cpuirq_restore_lc()

使用局限

关闭与开启CPU总中断开关需要确保不会发生嵌套，例如下面的嵌套代码是 错误 的，因为临界区1结束时会意外地将CPU中断打开， 从而无法保证临界区0的安全性：

void func1(void)
{
        xwos_cpuirq_disable_lc();
        /* ... 临界区1 ... */
        xwos_cpuirq_enable_lc();
}

void func0(void)
{
        xwos_cpuirq_disable_lc();
        /* ... 临界区0 ... */
        func1(); /* 错误！！！发生嵌套 */
        /* ... 临界区0 ... */
        xwos_cpuirq_enable_lc();
}

因此，当关闭中断的代码发生嵌套时，需要保存与恢复CPU总中断开关，上面的例子可以更正为：

void func1(void)
{
        xwreg_t cpuirq;

        xwos_cpuirq_save_lc(&cpuirq);
        /* ... 临界区1 ... */
        xwos_cpuirq_restore_lc(cpuirq);
}

void func0(void)
{
        xwreg_t cpuirq;

        xwos_cpuirq_save_lc(&cpuirq);
        /* ... 临界区0 ... */
        func1();
        /* ... 临界区0 ... */
        xwos_cpuirq_restore_lc(cpuirq);
}

外设中断

中断配置

中断配置通常包含中断优先级等配置。

用户通过 xwos_irq_request() 申请中断时，需要给出中断的配置， 也可通过 xwos_irq_cfg() 重新配置中断。 当需要获取中断配置时，可通过 xwos_irq_get_cfg() 。

为某个芯片移植XWOS时，需要在操作系统移植实现层(XWOSIMPL)的 xwosimpl_soc_irq.h 文件中定义 中断配置 结构体：

struct soc_irq_cfg;

各种架构的实现举例：

  • ARMv6m、ARMv7m的定义如下：
struct soc_irq_cfg {
        struct cortexm_nvic_irq_cfg irqcfg;
};
  • RISC-V/nuclei/gd32v的定义如下：
struct soc_irq_cfg {
        xwu8_t level;
        xwu8_t priority;
        xwu8_t trig;
        xwu8_t shv;
};
  • PowerPC/e200z0h/mpc560xb的定义如下：
struct soc_irq_cfg {
        xwu8_t priority;
};

中断数据

中断数据是位于 中断数据表 struct soc_idvt 中的数据，与中断一一对应。

用户可通过 xwos_irq_get_data() 获取中断的数据。

为某个芯片移植XWOS时，需要在操作系统移植实现层(XWOSIMPL)的 xwosimpl_soc_irq.h 文件中定义 中断数据 结构体：

struct soc_irq_data;

各种架构的实现举例：

  • ARMv6m、ARMv7m的定义如下：
struct soc_irq_data {
        void * data;
};
  • RISC-V/nuclei/gd32v的定义如下：
struct soc_irq_data {
        void * data;
};
  • PowerPC/e200z0h/mpc560xb的定义如下：
struct soc_irq_data {
        void * data;
};

获取当前代码的中断号

XWOS提供CAPI xwos_irq_get_id() 可以获取当前中断的中断号。 如果不在中断内使用此CAPI，将返回错误码：

  • -ETHDCTX ：当前上下文为线程
  • -EBHCTX ：当前上下文为中断底半部

因此，此CAPI还可用于 判断上下文 。

为某个芯片移植XWOS时，需要操作系统移植实现层(XWOSIMPL)中给出函数的实现方法：

  • 源文件： xwosimpl_irq.h/xwosimpl_irq.c
    • xwospl_irq_get_id() ：对应于 xwos_irq_get_id()

外设中断的其他CAPI

随着技术发展，现代芯片的供应商基本都提供了固件库，其中会封装操作外设中断的方法。因此 外设中断的CAPI 基本上不会被用到，趋近于 过时 。

  • xwos_irq_request() ：申请中断
  • xwos_irq_release() ：释放中断
  • xwos_irq_enable() ：开启中断
  • xwos_irq_disable() ：关闭中断
  • xwos_irq_save() ：保存中断的开关，然后将其关闭
  • xwos_irq_restore() ：恢复中断的开关
  • xwos_irq_pend() ：挂起中断标志
  • xwos_irq_clear() ：清除中断标志
  • xwos_irq_tst() ：测试中断是否挂起
  • xwos_irq_cfg() ：配置中断
  • xwos_irq_get_cfg() ：获取中断的配置
  • xwos_irq_get_data() ：获取中断的数据

为某个芯片移植XWOS时，需要操作系统移植实现层(XWOSIMPL)中给出函数的实现方法：

  • 源文件： xwosimpl_irq.h/xwosimpl_irq.c
    • xwospl_irq_request() ：对应于 xwos_irq_request()
    • xwospl_irq_release() ：对应于 xwos_irq_release()
    • xwospl_irq_enable() ：对应于 xwos_irq_enable()
    • xwospl_irq_disable() ：对应于 xwos_irq_disable()
    • xwospl_irq_save() ：对应于 xwos_irq_save()
    • xwospl_irq_restore() ：对应于 xwos_irq_restore()
    • xwospl_irq_pend() ：对应于 xwos_irq_pend()
    • xwospl_irq_clear() ：对应于 xwos_irq_clear()
    • xwospl_irq_tst() ：对应于 xwos_irq_tst()
    • xwospl_irq_cfg() ：对应于 xwos_irq_cfg()
    • xwospl_irq_get_cfg() ：对应于 xwos_irq_get_cfg()
    • xwospl_irq_get_data() ：对应于 xwos_irq_get_data()
